# Локальный запуск

Этот проект запускается полностью локально. Файлы не отправляются на внешний сервер: web-интерфейс поднимается на твоем компьютере и слушает адрес `127.0.0.1`.

## 1. Подготовка окружения

Перейди в папку проекта:

```bash
cd "/Users/stepazilin/Desktop/dedsec style"
```

Создай виртуальное окружение:

```bash
/usr/bin/python3 -m venv .venv39
```

Установи зависимости:

```bash
.venv39/bin/python -m pip install -r requirements.txt
```

## 2. Запуск localhost

```bash
.venv39/bin/python dedsec_converter.py --gui
```

В терминале появится адрес:

```text
http://127.0.0.1:8765
```

Открой его в браузере. Если порт `8765` занят, программа попробует следующий свободный порт.

## 3. Как пользоваться страницей

1. Нажми кнопку выбора файла или вставь полный путь к файлу.
2. Выбери алгоритм дизеринга.
3. Настрой интенсивность glitch.
4. Включи или выключи эффекты.
5. Нажми `EXECUTE CONVERSION`.

После обработки появится ссылка на готовый файл и предпросмотр.

## 4. Запуск двойным кликом на macOS

Открой `run_dedsec_converter.command`. Скрипт:

- переходит в папку проекта;
- создает `.venv39`, если его нет;
- ставит зависимости, если они не установлены;
- запускает `dedsec_converter.py --gui`.

## 5. Частые проблемы

### Ошибка `No module named cv2`

Зависимости не установлены. Выполни:

```bash
.venv39/bin/python -m pip install -r requirements.txt
```

### Ошибка Tkinter на macOS

В проекте больше не используется Tkinter для основного интерфейса. Запускай:

```bash
.venv39/bin/python dedsec_converter.py --gui
```

Откроется браузерный localhost-интерфейс.

### Браузер не открылся автоматически

Открой адрес вручную:

```text
http://127.0.0.1:8765
```

## 6. Остановка сервера

В терминале, где запущена программа, нажми `Ctrl+C`.
